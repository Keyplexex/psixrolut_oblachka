### Создание единой кросс-провайдерной сервисной модели
Рассмотрим кросс-провайдерную классификацию облачных сервисов

Во второй лабораторной работе мы продолжаем использовать выбранный в первой лабораторной вариант классификации облачных сервисов, но расширяем его на несколько облачных провайдеров. В качестве примера были рассмотрены Amazon Web Services (AWS) и Microsoft Azure.
Основная идея второй лабораторной заключается в том, чтобы научиться классифицировать облачные сервисы независимо от вендора, сохраняя единую логику и иерархию.

### Если рассматривать AWS и Azure в рамках сервисных моделей, распределение будет следующим
#### К IaaS относятся сервисы, предоставляющие базовые вычислительные и хранилищные ресурсы, где пользователь управляет архитектурой и способом использования ресурсов:

- Amazon S3 / Azure Blob Storage — объектное хранилище данных
- Amazon Glacier / Azure Archive Storage — архивное хранилище с низкой стоимостью
- Azure Virtual Machines — виртуальные машины различных серий

#### PaaS-сервисы предоставляют готовую платформу, снимая с пользователя необходимость управления инфраструктурой:

- Amazon Redshift / Azure Analysis Services — облачные аналитические хранилища данных
- AWS Directory Service / Azure Managed Active Directory — управление каталогами и учетными записями
- Amazon SNS / Azure Notification Services — рассылка уведомлений
- AWS CodePipeline / Azure DevOps Pipelines — автоматизация сборки и деплоя
- Azure Data Factory — интеграция и обработка данных

#### К SaaS относятся готовые сервисы, предоставляющие функциональность через API:

- Amazon Translate — машинный перевод текста
- Amazon Transcribe — распознавание речи
- Amazon Polly — синтез речи
- Amazon ML / Azure ML — сервисы машинного обучения

В отличие от первой лабораторной работы, во второй лабораторной наблюдается смещение распределения в сторону платформенных сервисов.  Это связано с добавлением сервисов Azure, таких как Data Factory, Analysis Services, Managed Databases и сервисов аналитики и интеграции данных.

Также увеличилась доля Analytics и AI-сервисов, так как во второй лабораторной учитываются управляемые аналитические и интеллектуальные сервисы обоих провайдеров, а не только базовые инфраструктурные ресурсы.
